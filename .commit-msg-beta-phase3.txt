beta: wire CONSENSUS_V1 as first registered beta model (Phase 3)

Activates the beta framework from Phases 1–2 with the first real model.
CONSENSUS_V1 is a shadow copy of the existing ConsensusStrategy that runs
in the beta sandbox, generating simulated trades for isolated evaluation
without touching the live book.

Enabled via env: CONSENSUS_V1_BETA_ENABLED=true on Render. Default off.
Nothing changes on Render until that env var is set.

What changed:
- bot.py: new CONSENSUS_V1_BETA_ENABLED env flag alongside LAG/CONSENSUS/
  SNIPER enables. Default "false".
- bot.py: KalshiBot.__init__ conditionally constructs a second
  ConsensusStrategy instance with is_beta=True, beta_model_id=
  "CONSENSUS_V1", same max_stake as live (so simulated P&L is
  apples-to-apples with what the disabled live Consensus would have done),
  and appends to self.beta_strategies.
- bot.py: run_once propagates self.consensus.last_result /
  last_result_time to every beta ConsensusStrategy instance after the
  live update_previous() call. Avoids double-polling the settled-markets
  API and keeps the two views of "previous result" in sync. Previous
  result is objective ground truth — no experimental value in each beta
  instance querying it independently.
- bot.py: startup banner lists registered beta model_ids so operators
  can confirm from Render logs whether the env var took effect.

Why CONSENSUS_V1 first (and not Expiry Decay):
- Validates Phase 1+2 plumbing against real live traffic with a known
  strategy whose parameters and edge profile are already understood.
- The live Consensus has been off for a week after going net-negative;
  beta is the right place to re-evaluate the v1.2 filters under current
  market conditions without risking capital.
- Tiny scope — ~25 LOC of production code. If anything is wrong in the
  framework, we find it here before committing to a larger new strategy
  build.

Deployment:
1. Push this commit.
2. On Render → Environment, add CONSENSUS_V1_BETA_ENABLED=true.
3. Redeploy. Startup logs should show "BETA models (1): CONSENSUS_V1".
4. Visit /beta on the dashboard — CONSENSUS_V1 card appears with Active
   status, zero fires, waiting for its first simulated trade.
5. Within the first cycle or two of matching market conditions, the
   model will start firing into reconcile_archive / bot_trades.json with
   is_beta=True + beta_model_id="CONSENSUS_V1".

Isolation invariants re-verified:
- Env off → self.beta_strategies is []
- Env on → single CONSENSUS_V1 instance, is_beta=True,
  beta_model_id="CONSENSUS_V1", stake=max_stake, independent cooldown +
  last_trade_time state
- Previous-result sync copies live → beta without writing back
